Error reporting

Zero-utilization BOM inputs. Component lines from the trailing 7 days of completed production runs where required qty was > 0 but operator-entered actual qty is 0 — typically meaning the operator didn't log component consumption, which breaks traceability for QA and unit costing. Pulled live from Cin7's Production Run data, so it catches both operator-entered zeros and lines the inventory-movement report would silently drop.

Production output

Trailing 7 days of completed production runs grouped by product category. Pulled from Cin7's Production Run API so transfers and stock adjustments are excluded — only what came out of manufacturing operations. Shares the same daily sync as the Error reporting tab.

Manufacturing yield

60-day trended yield per unit-op stage. Yield% = kg out / kg in, computed per completed Production Run and aggregated weekly. Stage assignment is driven by output SKU — admins can edit the SKU lists on the Yield Setup tab.
